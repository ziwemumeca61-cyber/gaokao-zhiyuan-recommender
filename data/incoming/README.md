# 原始数据投放区（data/incoming/）

把**其他省份的原始录取数据**（Excel/CSV，建议先压成 .zip）放到这个文件夹，
用 GitHub Desktop 提交并推送即可。我会把它导入、归一化后合并进 `data/real/`，
再把这里的原始文件清理掉，保持仓库精简。

## 怎么放（GitHub Desktop）

1. 把原始文件压成 zip（录取数据是文本，压缩后通常能小 5~10 倍）。
2. 把 zip 放进本文件夹 `data/incoming/`。
3. GitHub Desktop 里：填一句提交说明 → Commit → **Push origin**。
   - 网页拖拽才有 30MB 限制；Desktop 推送单文件可到约 100MB（50MB 以上有黄字警告但能传）。
4. 告诉我：是哪个省、文件名大概是什么。

## 文件大小怎么办

- **压缩后 < 100MB**：直接按上面推送即可。
- **压缩后仍 > 100MB**：按省/按批拆成多个 zip 分次推；或告诉我，我开 Git LFS。

## 数据里最好包含的列（有就行，列名不强求，导入器会自动识别别名）

- 院校名称、城市、层次（985/211/双一流/普通）
- 专业名称、（可选）学科门类
- 年份、科类（物理/历史/综合）、最低分、**最低位次**、（可选）招生计划数

> 位次最关键（推荐的核心依据）；缺门类/计划数不影响主流程，会自动兜底。
